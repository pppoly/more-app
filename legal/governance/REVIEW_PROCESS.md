# REVIEW_PROCESS v1.0 (2025-11-30)
1. 目的
1.1 法務文書改訂の承認フローを明確化し、APPI・電気通信事業法・特商法・消費者契約法への適合を担保する。

2. 役割
2.1 プロダクト：要件整理、UI文言整合。
2.2 運営：リスク/運用可否判断、告知計画。
2.3 法務：法令適合性、Stripe Connect 契約整合。
2.4 セキュリティ/リスク：風営・反社・未成年保護・外部送信規律確認。

3. 承認ステップ
3.1 ドラフト → 運営レビュー → 法務レビュー → リリース承認 → 公開。
3.2 重大変更（料金/責任/データ処理/支払/未成年/危険行為）は必ず法務承認。
3.3 公開後、UI/メール/アプリ内通知で周知。

4. 記録
4.1 `CHANGELOG.md` に改訂日・担当・概要・影響範囲。
4.2 差分と旧版を 2 年以上保管。

5. 例外処理
5.1 緊急リスク時は「Exception Handling」を適用し、事後 5 営業日以内に正式版をリリース。
