# VERSIONING v1.0 (2025-11-30)
1. 方針
1.1 バージョンは `vX.Y` を採用し、主要改訂は X、副次改訂は Y で管理する。
1.2 すべての法務文書にバージョンと改訂日を明記する。

2. 適用範囲
2.1 利用規約・プライバシー・支払/返金・主催者規約・リスク/ガバナンス文書を含む。
2.2 多言語版（JA/EN/ZH）は同一バージョンで同期。

3. 改訂プロセス
3.1 「REVIEW_PROCESS.md」に従い、製品→運営→法務の承認を経る。
3.2 改訂後は `CHANGELOG.md` に記録し、UI/通知で周知。

4. バックポート
4.1 重大リスクや法令対応はマイナーバージョンにもバックポート可能。
4.2 バックポート時は注記を残し、影響範囲を明示。
